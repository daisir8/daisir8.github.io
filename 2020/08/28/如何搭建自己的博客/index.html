title:基于hexo在windows下建立个人博客

博客是继MSN、BBS、ICQ之后出现的第4种网络交流方式，现已受到大家的欢迎，是网络时代的个人“读者文摘”，
是以超级链接为入口的网络日记，它代表着新的生活、工作和学习方式。
MSN（信息门户网站）,BBS（网络论坛），ICQ（即时通讯软件）

#整体思路：我们的目的是制作一个博客，用于网络交流，分享自己的想法，发表自己的文章。肯定少不了别人通过网络访问
你，你需要在本地拥有一个博客文件，然后再推送它到远端，这样人们就可以来访问了。

##第一步：下载Node.js

Node.js下载地址

下载LTS版（长期支持版）

##第二步：下载hexo

win+r 输入cmd打开终端

输入npm install -g cnpm –registry=https://registry.npm.taobao.org，安装镜像包

（这一步的操作是为了下载hexo快一点）

输入cnpm install -g hexo-cli 安装hexo

##第三步：下载git

git下载地址

同时在cmd中安装git插件：cnpm install –save hexo-deployer-git

##第四步：准备工作做得差不多了，现在开始建立本地博客文件夹

终端（cmd）中在d盘建立blog文件夹（在cmd中 输入 mkdir blog），然后进入文件夹（在cmd中输入cd blog），
准备工作完成之后的第一步操作：hexo init（初始化hexo）

第二步：输入hexo s（启动hexo）
这个时候cmd中会蹦出一个localhost：4000，你输入到浏览器中就可访问了，不过你一退出这个hexo s 就无法访问了

你就能看到一个最初版的hexo

第三步：我们肯定不能用初始化的东西，cmd 中输入hexo n “你新文章的名字”，新建文章，hexo n命令生成的新文件一般都在blog文件夹下的
source/_posts/，所以我们要cd它们一下再次进入，然后notepad 新文章的文件名或者vim 新文章的文件名，对文章进行编辑。（我有一个更好的办法
根本不需要在cmd中操作，直接打开文件夹，用记事本打开，然后按照Markdown的格式改写就好了），然后保存退出。

第四步：然后就是推送它到远端了，退回到刚刚的blog目录，cd …/…
此时notepad一下_config.yml这个文件，也就是编辑一下这个文件，改动一下你这个本地博客文件夹推送到远端的方式
我们这里用git（因为git不要钱哈哈，不然你租个服务器，随着时光的流动，钱也就溜走了。）

具体操作：notepad一下_config.yml这个文件之后，往下翻，看到了一个deployment（配置的意思），下面有个
type:” 此刻把引号删掉，改成type: git，因为我们要用git嘛，记得冒号之后打空格，别连起来。
下面再加
repo：输入你在github中新建仓库的网址（有关git的知识我会在后面讲）
branch：master

然后推送它到远端三步走：hexo clean——hexo g——hexo d

有时候会出现一个tell me who you are 的错误，这是因为你的git没有登录，此时输入

git config –global user.name github里的昵称
git config –global user.email github里的昵称@github.com

然后它会出现一个让你登录github的界面，登录就好了

然后在浏览器中输入你在github中新建仓库的网址（一般是仓库名+github.io），就可以登录看你的博客了，不仅你可以看，别人也可以哦。

更换博客主题我略讲一下

cmd 中git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

克隆好后修改_config.yml

theme：那一栏改为yilia

然后就是老三步：hexo clean ——hexo g ——hexo d

在浏览器中输入你在github中新建仓库的网址（一般是仓库名+github.io），就可以登录看你的博客了

git小知识
git是一种分布式版本控制系统，我的理解为分布式项目管理系统，开发者可以通过git clone 在
本地机器上拷贝一个完整的git仓库（通过git clone 获得别人的项目，就像你上面用别人制作的yilia主题一样）
而你可以通过git，可以管理的你项目，放置不同的版本的项目，以后能用的着。
